documentation for the videodrömm project

Goal

Create multiplatform applications (win, mac, lnx, android, ios) to do VJing/installations

Structures

Pure C++ and js

Engines

openFrameworks and Cinder frameworks will be used for the C++ part, based on the pure C++ structures. For javascript, use angularjs, nodejs, mongodb, threejs but that's to be determined later.

Network

OSC and webSockets will be used. The goal is to have a standalone renderer, which can be controlled remotely with

  • TouchOsc on mobile/tablet: we stick to TouchOSC default messages like "/1/fader1"
  • LiveOSC for Ableton: messages likes "/live/meter"

Applications

Several application will be developed

  • Videodrömm visualizer(C++): standalone renderer for projections/installations
  • Videodrömm server(nodejs): webSockets server for multiple websocket clients, routes the json messages from OSC, MIDI, etc
  • Videodrömm livecoding(C++): editor livecoding of glsl shaders, when they compile successfully, they are sent to the server via webSockets
